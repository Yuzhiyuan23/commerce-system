## 数据库设置步骤

### 1. 创建数据库

在phpMyAdmin中执行：
```sql
CREATE DATABASE IF NOT EXISTS commerce_system
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;
```

### 2. 导入数据库结构和样例数据

打开 `ops/mysql/init-complete.sql` 文件，复制全部内容到phpMyAdmin的SQL执行窗口中执行。

这个SQL文件包含：
- 所有数据表结构（兼容MySQL 5.5+）
- 8个商品分类
- 24个样例商品
- 对应的SKU和库存数据

### 3. 验证数据导入

执行以下SQL检查数据：
```sql
-- 查看分类
SELECT * FROM product_categories;

-- 查看商品数量
SELECT COUNT(*) FROM products;

-- 查看SKU数量
SELECT COUNT(*) FROM product_skus;
```

## 启动服务

### 1. 确保MySQL已启动

XAMPP控制面板中启动MySQL服务。

### 2. 启动后端

```bash

cd target
java -jar commerce-system-backend-0.0.1-SNAPSHOT.jar
```

或者使用IDE直接运行 `CommerceSystemApplication`。

后端将运行在 http://localhost:8080

### 3. 启动前端

```bash
cd frontend/next-app
npm run dev
```

前端将运行在 http://localhost:3000

## 系统架构

### 技术栈
- **后端**: Java 21 + Spring Boot 4.0.6 + MyBatis-Plus + Spring Security
- **前端**: Next.js 15 + React 19 + TypeScript 5.8 + Tailwind CSS 4
- **数据库**: MySQL 5.5+ (utf8mb4_unicode_ci)
- **缓存**: Redis (可选，默认可禁用)
- **推荐引擎**: Gorse (可选，默认可禁用)
- **邮件**: Spring Mail + MailHog (开发环境)
- **对象存储**: 阿里云OSS (可选)

### 用户角色
1. **CUSTOMER** - 普通客户：浏览商品、下单、查看订单
2. **SALES** - 销售人员：处理订单、发货、查看销售数据
3. **ADMIN** - 管理员：所有权限 + 用户管理、商品管理、数据分析

### 推荐算法
1. **首页推荐**: 基于用户行为的协同过滤（Gorse）+ 热门商品
2. **详情页推荐**: 基于物品的协同过滤 + 同类商品
3. **兜底策略**: 当推荐服务不可用时，返回热门商品

### 数据分析
1. **用户画像**: 地域分布、购买力分层、品类偏好、复购率
2. **销售趋势**: 日/周/月粒度销售统计 + 移动平均线
3. **商品排行**: 销量排行、销售额排行
4. **异常检测**: 基于标准差的销售额异常检测

## 配置文件说明

### 后端配置 (`backend/src/main/resources/application.yml`)

关键配置项：
```yaml
# 数据库连接
spring.datasource.url: jdbc:mysql://localhost:3306/commerce_system
spring.datasource.username: commerce
spring.datasource.password: commerce123

# Flyway已禁用（使用手动SQL导入）
spring.flyway.enabled: false

# Redis（可选）
spring.data.redis.host: localhost

# 推荐系统（可选）
commerce.recommendation.gorse.enabled: false

# 阿里云OSS（可选）

oss.bucket: your-bucket
oss.access-key-id: your-key
oss.access-key-secret: your-secret
```

## 常见问题

### 1. 后端启动失败

检查：
- MySQL是否已启动
- 数据库 `commerce_system` 是否已创建
- 数据库用户名密码是否正确

### 2. 前端无法连接后端

检查：
- 后端是否运行在8080端口
- `.env.local` 中的后端地址是否正确
- 是否有防火墙阻止连接

### 3. 图片上传失败

如果没有配置OSS，上传功能会失败。可以：
- 配置阿里云OSS参数
- 或使用本地文件存储（需要修改代码）

### 4. 推荐系统不工作

推荐系统依赖Gorse服务，默认已禁用。如需启用：
1. 启动Gorse服务（Docker）
2. 设置 `commerce.recommendation.gorse.enabled: true`

不启用时，系统会返回热门商品作为推荐。

## 测试账号

系统首次启动时，需要手动创建用户。可以通过注册功能创建普通用户，然后直接在数据库中修改角色：

```sql
-- 将用户设为管理员
INSERT INTO user_roles (user_id, role_id)
SELECT u.id, r.id
FROM users u, roles r
WHERE u.email = 'your@email.com'
  AND r.code = 'ADMIN';
```
